What Is Flexbox?

"The CSS3 Flexible Box, or flexbox, is a layout mode providing for the arrangement of elements on a page such that the elements behave predictably when the page layout must accommodate different screen sizes and different display devices."

- MDN (Mozilla Developer Network)

It's important to note what Flexbox is not. It is not a grid system (CSS grids is for that) and although it can be used for page layouts, it works best on building elements responsively.

Flexbox is a CSS layout mode

  • Block layout: for stacked elements
  • Inline layout: for text
  • Table layout: for (you guessed it) tables
  • Positioned layout: for elements without much interaction with other elements
  • Flexbox layout: designed for coding responsive elements that can be resized smoothly
To understand Flexbox as a CSS layout, let's look at other modern CSS layouts and what they are used for.

Floats

...were originally meant to float text around an image.


...that's it.

Floats are not a layout system at all but developers found how useful they could be in laying out elements inline to each other and hacked it to be a layout system similar to how tables used to be used for full page layouts.

Why we love Flexbox

  • Can replace floats to align elements smoothly
  • Automatic distribution of extra space
  • Vertical centering!!
  • Less code! More efficiency and less calculations of paddings, margins, widths

Flexbox Basics

One of the first things to learn about flexbox is the vocabulary. When you are declaring a flexbox element it is known as the flex container. Any direct child element inside that element will automatically be defined as a flex item and inherit any flex properties that are declared on the container level. The Parent-child relationship is one of the best parts and allows you to keep your code D.R.Y.

The Flexbox Axes

Imagine each axis as an invisible line along which the child elements (aka flex items) are aligned. The main axis will depend on if the flex-direction is row or column whether it is left-to-right or up-to-down. The cross axis will be the opposite of the main axis. There are specific properties to control how the child elements are arranged along each axis which we will get into in a sec.

Let's Get Started!

First, we'll look at properties for the flex container

                    .flex-container {
                        display: flex;
                    }
Several things are happening by default all with one line of code. Defaulting to flex-direction: row; and justify-content: flex-start;

Flex-Flow:

A shorthand for 2 flexbox properties


                    .flex-container {
                        display: flex;
                        flex-flow: row nowrap;
                        /* is the same as:
                           flex-direction: row;
                           flex-wrap: nowrap;
                        */
                    }
  • flex-direction: establishes the main axis
    values: row(default) | row-reverse | column | column reverse
  • flex-wrap: allows items to wrap to another line
    values: nowrap(default) | wrap | wrap-reverse

Justify-content:

Alignment along the main axis and behavior of extra space not used by the content of the flex-items

values: flex-start (default) | flex-end | center | space-between | space-around

                    .flex-container {
                        display: flex;
                        flex-flow: row wrap;
                        justify-content: space-between;
                    }

Align-items:

Alignment along the cross axis and behavior of extra space not used by the content of the flex-items

values: stretch (default) | flex-start | flex-end | center | baseline


                    .flex-container {
                        display: flex;
                        flex-flow: row nowrap;
                        justify-content: space-between;
                        align-items: center;
                    }
These 4 properties are the meat & potatoes of flexbox and with these you can already start making flexible elements which we will practice next.

What about Flex Items?

Now, we'll look at properties you can apply to individual flex items to override the flex properties applied to the parent element.

Order:

  • Controls the order in which the items appear
  • Does NOT change the source order
  • All items default to order: 0
value: (any positive or negative integer)

                    .item1 {
                        order: 2;
                    }

                    .item2 {
                        order: -4;
                    }
Make sure not to use on text or content that the user might want to copy/paste because the source order is not changing.

Align-self:

Overrides the align-items property set on the flexbox container

value: flex-start | flex-end | center | stretch | baseline

                    .flex-container {
                        align-items: center;
                    }

                    .item1 {
                        align-self: flex-start;
                    }

Auto Margins and Flexbox

Setting a margin: auto on a flex item can be used to position the flex item using the available extra space.


                    .item2 {
                        margin-left: auto;
                    }

Auto Margins and Flexbox


                    .item1 {
                        margin-left: auto;
                    }

Flexibility

"The defining aspect of flex layout is the ability to make the flex items “flex”, altering their width/height to fill the available space in the main dimension. A flex container distributes free space to its items proportional to their flex grow factor, or shrinks them to prevent overflow proportional to their flex shrink factor."
- W3C
This is definitely the most difficult aspect of flexbox but its also one of the most powerful concepts. The extra space is distributed proprotional, as seen in space-between or space-around but you can control on individual flex items.

3 Flex properties

  • flex-grow: specifies the flex grow factor of a flex item. Default is 0.
  • flex-shrink: specifies the flex shrink factor of a flex item. Default is 1.
  • flex-basis: specifies the initial size of the flex item, before any available space is distributed according to the flex factors. Default is 0.
Depending on how much space the content along an axis takes up, the extra space will be distributed equally depending on how you specify the flex-container. If you have a 500px container with 2 100px wide items ther extra 300px will equally be divide among the items if they are both flex: 1. If one item has flex grow: 1 it will take up 20px extra instead of 10px.

A closer look at how the flex proportional values work

Auto will size the flex item based on its contents and distribute extra space around it. 0 will apply the extra space to the entire flex item regardless of content.

Flex Shorthand

Recommended for setting all of the flex properties

                    .flex-item {
                        flex: 0 0 auto;
                    }

                    /* is equal to */
                    .flex-item {
                        flex-grow: 0;
                        flex-shrink: 0;
                        flex-basis: auto;
                    }

Basic Flex Values

There are 4 default values for flex properties:

  • flex: initial; /* (Equivalent to flex: 0 1 auto) */
  • flex: auto; /* (Equivalent to flex: 1 1 auto) */
  • flex: none; /* (Equivalent to flex: 0 0 auto) */
  • flex: 'integer'; /* (Equivalent to flex: 'integer' 1 0) */

For more in-depth information on the four chosen scenariors, visit this section of the Flexbox spec
-Initial: Makes the flex item inflexible when there is positive free space, but allows it to shrink to its minimum size when there is insufficient space. - Auto: Sizes the item based on the width/height properties, but makes them fully flexible, so that they absorb any free space along the main axis. - None: This is similar to initial, except that flex items are not allowed to shrink, even in overflow situations. - Positive Integer: Makes the flex item flexible and sets the flex basis to zero, resulting in an item that receives the specified proportion of the free space in the flex container.
